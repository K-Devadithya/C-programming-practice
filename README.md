# C Programming Practice Repository

Welcome to my C programming practice repository! This project serves as a structured collection of fundamental algorithms, mathematical computations, and data structure implementations designed to build a rock-solid foundation in procedural programming.

---

## Folder Structures & Concepts Learned

### 1. basic-math

This directory handles fundamental arithmetic operations, algorithmic logic systems, and conditional branching logic using scalar variables.

* **Programs Included:**
* `Calculator.c`: A menu-driven tool using multi-case evaluation blocks.
* `LCM.c`: Finds the Lowest Common Multiple using common factor evaluation loops.
* `Leap_year.c`: Processes specific century and year divisibility rules.
* `Number_sign.c`: Evaluates whether a target number is positive, negative, or zero.
* `Sigma.c`: A loop-based accumulator calculating the sequence summation of $N$ natural numbers.
* `Sum_of_digits.c`: Extracts individual digit values mathematically and compiles their total sum.


* **Concepts Mastered:**
* Conditional expressions (`if`, `else if`, `else`).
* Control flow switching (`switch-case` statements) for menu designs.
* Arithmetic operators, mathematical formulas, and conditional logic flattening using logical AND/OR (`&&`, `||`).



---

### 2. number-programs

This folder focuses on intermediate numerical analysis, loops, and mathematical algorithms used to deconstruct, inspect, or generate complex integer properties.

* **Programs Included:**
* `Armstrong_number_check.c`: Validates numbers whose digits raised to the power of the total number of digits equal the original number.
* `Digit_counter.c`: Utilizes sequential integer division loops to determine value scale.
* `Fibonacci_series.c`: Computes additive sequence chains up to a target boundary.
* `Multiplication_table.c`: Generates algebraic product rows using controlled loops.
* `Palindrome_check.c`: Reverses digit order to evaluate symmetry against original values.
* `Prime_numbers.c`: Implements optimal square-root boundary testing ($\le \sqrt{i}$) to generate prime distributions.


* **Concepts Mastered:**
* Looping execution controls (`for`, `while`, and `do-while` loops).
* Digit manipulation techniques using modular arithmetic (`% 10`) and truncation division (`/ 10`).
* Optimizing nested loops and flag-state triggers (`break` checks) for performance-critical math pipelines.



---

### 3. arrays-and-matrices

This directory moves beyond singular memory storage blocks into sequential data collections, handling strings (character arrays) and multi-dimensional grid elements.

* **Programs Included:**
* `Matrices_2D.c`: Handles matrix calculations, grid processing, or row-column alignments.
* `word_counter.c`: Processes character arrays through custom state logic to filter, identify, and count alphanumeric spaces/words.


* **Concepts Mastered:**
* One-dimensional arrays and string null-termination mechanics (`'\0'`).
* Dynamic memory allocation strategies (`malloc()`, tracking sizes, and handling allocation scaling).
* Multi-dimensional array structures (2D grids) utilizing nested index structures (`array[i][j]`).



---

### 4. pattern-programs

This directory focuses on visual geometry output generation, requiring strict manipulation of nested coordinate loops.

* **Programs Included:**
* `Triangle.c`: Renders right-angled layout models using ascending symbols.
* `Rev_triangle.c`: Generates inverted spatial pattern arrays.


* **Concepts Mastered:**
* Nested coordinate nesting grids ($X$ and $Y$ axis output syncing).
* Controlling text formatting streams (`\n`, `\t`, padding configurations).
* Developing structural loop boundaries where outer iteration counters tightly govern internal loop conditions.

## General Skills Developed Across Projects

* **Memory Safeguards:** Learned to preserve core state parameters using temporary copies and cleanly clearing dynamically allocated pointers using `free()`.
* **Terminal Environment Agnosticism:** Replaced old compiler libraries (like `<conio.h>`) with standard streams (`getchar()`, loop buffer clears) to ensure code compiles anywhere.
* **Defensive Code Engineering:** Implemented check systems to neutralize execution hazards like division-by-zero occurrences and bad user input flags.
